# Commerce Registry Protocol

The Commerce Registry Protocol is an open, versioned specification for registering commerce gateways, proving domain control, and exposing agent-facing discovery APIs. Any compliant implementation can interoperate — including the public registry at `https://registry.betterdata.co` and self-hosted deployments.

## In this documentation

- [Registration API](/docs/protocol/registration) — `POST /api/gateways` and related registration flows
- [Resolution](/docs/protocol/resolution) — how agents and resolvers look up gateways
- [.well-known specification](/docs/protocol/well-known) — discovery file format and trust semantics

For hands-on setup on a hosted or self-hosted gateway, start with [Register Your Gateway](/docs/registry/setup) and [Registry Overview](/docs/registry/overview).
